# Contributing

Thanks for your interest in contributing. We welcome issues and pull requests. By contributing, you agree to follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## Before you start

For bugs or features, please open an issue first. That way we can align and avoid duplicate work. If an issue already exists, you can comment that you plan to work on it.

## Reporting a bug

Open a [new issue](https://github.com/FluxonApps/llm-testrunner-lib/issues/new). Please include:

- Steps to reproduce
- What you expected vs what happened
- Your environment (e.g. Node version, OS) if relevant

## Suggesting a feature or change

Open a [new issue](https://github.com/FluxonApps/llm-testrunner-lib/issues/new) and describe the idea and why it would help.

## Pull requests

1. Fork the repo and clone your fork.
2. Create a branch from `main` for your change.
3. Run `npm ci` to install dependencies.
4. Make your changes. Then run:
   - `npm run lint`
   - `npm run test`
   - `npm run license-check`
   so everything passes before you push.
5. Push your branch and open a pull request against `main`.

Keep each PR focused on one change and describe what it does in the PR description.
